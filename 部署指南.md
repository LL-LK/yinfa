## 一、部署后端到 Railway

### 1. 准备工作

1. 确保代码已推送到 GitHub 仓库
2. 拥有 Railway 账号（https://railway.app）

### 2. 部署步骤

1. 登录 Railway 并点击 **"New Project"**
2. 选择 **"Deploy from GitHub repo"**
3. 选择你的仓库 `LL-LK/yinfa`
4. Railway 会自动检测 `railway.json` 配置

### 3. 配置环境变量

在 Railway 项目设置中添加以下环境变量：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `NODE_ENV` | `production` | 生产环境标识 |
| `PORT` | `8000` | 服务端口 |
| `ALLOWED_ORIGINS` | `*` | 允许的跨域来源 |

### 4. 配置持久化存储

Railway 默认文件系统是只读的，需要添加持久化卷来保存数据库：

1. 进入 Railway 项目 → **Settings** → **Volumes**
2. 点击 **"Add Volume"**
3. 设置 Mount Path 为 `/app/data`
4. 保存后重启服务

### 5. 获取部署URL

部署成功后，Railway 会提供一个 URL，例如：
```
https://yinfa-production-xxxx.up.railway.app
```

### 6. 验证部署

访问以下端点验证：
- 首页：`https://your-railway-url.up.railway.app/`
- 健康检查：`https://your-railway-url.up.railway.app/health`
- API：`https://your-railway-url.up.railway.app/api/categories`

## 二、配置微信小程序连接后端

### 1. 在小程序中设置API地址

打开微信开发者工具，在控制台执行：
```javascript
getApp().setApiUrl('https://your-railway-url.up.railway.app/api')
```

或者修改 `app.js` 中的默认地址：
```javascript
globalData: {
  apiBaseUrl: 'https://your-railway-url.up.railway.app/api',
  // ...
}
```

### 2. 配置服务器域名

在微信公众平台 → 开发管理 → 开发设置 → 服务器域名中，添加你的 Railway URL：
```
https://your-railway-url.up.railway.app
```

### 3. 开启不校验域名（开发阶段）

在微信开发者工具中：
1. 点击右上角 **"详情"**
2. 勾选 **"不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书"**

## 三、本地开发

### 后端启动

```bash
cd yinfa-ts/backend/server
npm install
npx ts-node src/index.ts
```

服务运行在 `http://localhost:8000`

### 小程序开发

1. 打开微信开发者工具
2. 导入项目：选择 `yinfa-ts` 目录
3. 确保后端服务已启动
4. 点击编译

### Web前端开发（可选）

```bash
cd yinfa-ts/frontend
npm install
npm run dev
```

## 五、常见问题

### 1. 后端连接失败

- 检查后端服务是否正常运行
- 检查 `apiBaseUrl` 配置是否正确
- 开发阶段确保已关闭域名校验

### 2. 数据库数据丢失

- Railway 无持久化卷时，重启会丢失数据
- 确保已添加 Volume 并设置 Mount Path 为 `/app/data`

### 3. 跨域错误

- 检查 `ALLOWED_ORIGINS` 环境变量配置
- 小程序需要在公众平台配置服务器域名
