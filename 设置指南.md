# 设置指南

## 环境准备

### 前端环境
- Node.js 16.0+
- npm 8.0+
- Vue 3
- TypeScript
- Vite

### 后端环境
- Node.js 16.0+
- npm 8.0+
- Express
- TypeScript
- SQLite

## 项目设置

### 1. 克隆项目

```bash
git clone <repository-url>
cd yinfa
```

### 2. 前端设置

1. 进入前端项目目录
```bash
cd yinfa-ts
```

2. 安装依赖
```bash
npm install
```

3. 启动开发服务器
```bash
npm run dev
```

4. 构建生产版本
```bash
npm run build
```

### 3. 后端设置

1. 进入后端项目目录
```bash
cd yinfa-ts/server
```

2. 安装依赖
```bash
npm install
```

3. 启动开发服务器
```bash
npm run dev
```

4. 构建生产版本
```bash
npm run build
```

## 微信小程序配置

1. 打开微信开发者工具
2. 导入项目：选择 `yinfa-ts` 目录
3. 修改小程序配置文件 `project.config.json`，确保正确设置
4. 在小程序中配置 `app.js` 文件，设置API基础URL：
   ```javascript
   apiBaseUrl: 'http://localhost:8000/api'
   ```
5. 编译运行小程序

## 数据库设置

后端使用SQLite数据库，无需额外配置。数据库会在首次启动时自动初始化并创建示例数据。

## 开发流程

1. 启动后端服务器：`cd yinfa-ts/server && npm run dev`
2. 启动前端开发服务器：`cd yinfa-ts && npm run dev`
3. 在浏览器中访问前端：`http://localhost:5173`
4. 测试API接口：`http://localhost:8000/api/categories`

## 常见问题

### 1. 后端启动失败

**问题**：`Error: Cannot find module 'sql.js'`
**解决方案**：确保已安装所有依赖，运行 `npm install`

### 2. 前端编译错误

**问题**：`Module not found: Error: Can't resolve 'vue-router'`
**解决方案**：确保已安装路由依赖，运行 `npm install vue-router`

### 3. API接口返回404

**问题**：API请求返回404错误
**解决方案**：确保后端服务器正在运行，并且API路径正确

### 4. 数据库初始化失败

**问题**：`Error: SQLITE_ERROR: no such table`
**解决方案**：删除 `server/data/shop.db` 文件，重新启动后端服务器

## 部署建议

### 前端部署
1. 构建生产版本：`npm run build`
2. 将 `dist` 目录部署到静态文件服务器

### 后端部署
1. 构建生产版本：`npm run build`
2. 使用 PM2 或其他进程管理工具启动服务：`pm2 start dist/index.js`

### 环境变量

在生产环境中，可以通过环境变量配置服务器端口：

```bash
# Linux/Mac
export PORT=8000

# Windows (cmd)
set PORT=8000

# Windows (PowerShell)
$env:PORT=8000
```
